In 26-125, Editor's Report for 26-007, the Editor wondered if, in addition
to noting that the real model is not IEEE 754, that we should add that
the integer model is not two's complement as it is symmetrical around
zero.

An edit is supplied.

Edit to 26-007r1

[423:4+] 17.4p3+ Numeric models

In Note 1's second sentence, after "In particular," insert:

"the integer model is not two's complement and "

so that the full sentence reads:

"In particular, the integer model is not two's complement and the
real model is not the same as any format defined by \theIEEEstd."
