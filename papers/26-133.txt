Subject: US-09 - Yet another edit for Allow I/O of Enumerator Names
Date: 2026-March-30

References: 26-007r1, 25-201r1, 25-126r1, 23-151r1

Paper 25-201r1. which supplied edits for the US-09 feature to allow
I/O of enumerator names, neglected to specify the behavior for
namelist input when the next effective variable is of enumeration type,
even though other text indicated it was allowed. Appropriate text
was provided for list-directed input.

An edit is provided to supply the missing text, similar to that for
list-directed input. Note that it is not necessary to add text for
namelist output as that is already covered by the reference to
list-directed output in 13.11.4.2.

Edit to 26-007r1
================

[337:35+]. 13.11.3.3p6+ Namelist input values

After p6 (integer or Enum type), insert a new paragraph:

"When the next effective item is of an enumeration type, the input form
of the input value is interpreted as if a Gw edit descriptor with a suitable
value of w were used."

==END==
