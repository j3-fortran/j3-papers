Subject: Edit for namelist input of logical values
Date: 2026-March-30

References: 26-007r1

13.11.3.3p5, Namelist input values, says:

"When the next effective item is of type logical, the input form of the input
value shall not include equals or value separators among the optional
characters permitted for L editing (13.7.3)."

This is all well and good, but never says that the input value shall be
suitable for L editing, unlike the paragraphs for other data types that all
specify the format edit descriptor to be used.

An edit is supplied to correct this.

Edit to 26-007r1
================

[337:33]. 13.11.3.3p5 Namelist input values

At the end of p5, add:

"The value in the input record is interpreted as if an Ln edit descriptor
with a suitable value of n were used."
