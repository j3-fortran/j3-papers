Subject: Edit for description of r*c in list-directed input
Date: 2026-March-30

References: 26-007r1, 26-125

In 26-125, the Editor's Report for 26-007, the Editor noted, in his
edits for US09:

"In this paragraph [13.10.3.1p2, List-directed input],
            "there is a nonblank character immediately after r*"
seems moot, as blanks are not permitted in the r*c form except as
permitted within c, and an undelimited character value cannot start
with a blank, so r*[blank] is the r* form not the r*c form.

This text was not new; it appeared in the published Fortran 2003
standard. JoR agrees that this is incorrect and an edit is
supplied.

Edit to 26-007r1

[331:32] 13.10.3.1p2, List-directed input

In the text "there is a nonblank character", delete the word "nonblank".
